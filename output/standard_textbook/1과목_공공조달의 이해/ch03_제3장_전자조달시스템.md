# 제3장 전자조달시스템의 이해 (표준교재 핵심)

> PDF p.127–230 · 박문각 Part 3

## 한눈에 보기

- **전자조달시스템**: ICT 기반 소싱·구매·지불 자동화. 공공조달에서는 **전자공공조달시스템 = 전자정부조달시스템**. E2E 수준 구현이 이상적이며, 우리나라 **KONEPS(나라장터)** 는 2002년 개통·UN·OECD 모범 사례.
- **유형**: 영국 분류 — **전자적 소싱**(e-Tendering·평가·협상·경매) vs **전자적 구매**(e-Catalogue·지불·송장). 지출 4사분면(기술적·전략적·인수·레버리지)에 따라 기능 중점이 달라짐.
- **나라장터**: 핵심(발주·입찰·계약·이행·종합쇼핑몰) + 지원(20여 개). 2025~ **디지털전환(DX)** 단계. 연계 플랫폼 — **종합쇼핑몰·혁신장터·벤처나라·디지털서비스몰·이음장터**.
- **목록정보**: **UNSPSC** 기반. 물품목록번호 = 물품분류번호(8자리) + 물품식별번호(8자리). 국내 **세부품명번호 10자리** 추가. 나라장터 계약·쇼핑몰 등록의 기준정보.
- **규모 지표 구분**: **156.8조** = 2002 대비 나라장터 **거래** 규모(4.3배) / **225.1조** = 2024말 **공공조달 전체** 규모 — 혼동 금지.

## 절별 핵심

### 제1절 전자조달시스템 개요

#### 1.1 정의·위상

| 구분 | 내용 |
|------|------|
| 정의 | 인터넷 등 ICT로 상품·용역 소싱·구매·지불을 자동화하는 전자적 프로세스 집합 |
| 공공 명칭 | 전자공공조달시스템 = 전자정부조달시스템 (동일) |
| 세계은행 | ICT로 공급업체와 조달관계 수행 |
| EU | 입찰공고·입찰서·평가·수주·주문·송장·지불 등 절차 전반 ICT 처리 |
| E2E | 초기엔 이상적 목표 → 현재 한·미·영·EU·OECD 등 사실상 End-to-End |
| KONEPS | 2002 서비스 개통, UN 전자정부 대상 등 수상 |

#### 1.2 유형·발전

- **영국(2005) 2유형**: 전자적 소싱 vs 전자적 구매
- **전자적 소싱**: 공급업체 DB, e-Tendering, e-Evaluation, e-Collaboration, e-Negotiation, e-Auction, 계약관리
- **전자적 구매**: Purchasing, Payment, Purchasing Card, e-Invoicing
- **한국 연혁**: 1970년대 FPDS → EDI(1980~90) → 2000.11 전자입찰 → **2002 KONEPS(나라장터)**. 2014까지 OECD 전원 1개 이상 플랫폼·대부분 의무화

#### 1.3 지출 4사분면 (Schapper)

| 유형 | 복잡성·위험 | 지출 | 전자조달 중점 |
|------|------------|------|--------------|
| **기술적** | 높음 | 낮음 | 계약·수요관리, 인소싱 |
| **전략적** | 높음 | 높음 | e-Tendering, EOI, 공개경쟁 (비중 최대) |
| **인수** | 낮음 | 낮음 | e-Catalogue, e-Payment, Workflow |
| **레버리지** | 낮음 | 높음 | e-Auction, e-Quoting, FA(기본협약) |

- **FA(기본협약)**: CPB가 사전 검증 공급업체와 기간·조건 합의 → 개별 조달 간소화. EU·한·미 **MAS**와 유사
- **DPS(동적구매시스템)**: 실시간 온라인 FA. EU는 DPS, 한·미는 **MAS**

#### 1.4 구성요소·도구

**발전 5단계** (UN): ①종이+일부 PC → ②메인프레임·PC·ERP·EDI → ③이메일·웹 → ④온라인 DB·카탈로그·웹 EDI → ⑤통합 플랫폼(현대적 E2E)

**주요 도구**

| 도구 | 핵심 |
|------|------|
| EDI | 배치 교환, 폐쇄망, 비용 높음 |
| e-Sourcing | 견적·입찰 비교, 공급업체 선정 |
| e-Tendering | REOI~ITB~RFP, 암호화·감사추적 |
| e-Auction / **역경매** | 순방향=높은 가격 / 역방향=낮은 가격. **국내 역경매 미채택** |
| e-Catalogue | 국내 → **종합쇼핑몰** |
| DPS | 공급업체 수시 가입, ~10일 입찰 |

#### 1.5 표준 기능·아키텍처

**표준 10기능**: 전자조달계획, 공급업체 위험평가, 전자게시/알림, e-Tendering, e-Evaluation/Award, e-Contract Management, e-Purchasing, Vendor Management, Auction

- **타임스탬프**: 내부 시계 의존 금지, 표준시 기반
- **FMIS 연계**: 국내 **dBrain**, **e호조**
- **조달 포털**: 공급업체·구매자 등록부, 입찰, 카탈로그, P-Card, e-Payment
- **PMIS**: 계획·계약관리·감사·보고 (포털과 연계)

#### 1.6 도입 성과 (6가지)

투명성·책임성 / 효율·속도 / 비용절감 / 공급업체 관리 / 입찰 접근성(중소기업) / 데이터·보고

### 제2절 국가종합전자조달시스템(나라장터)

#### 2.1 개요·발전

| 단계 | 시기 | 내용 |
|------|------|------|
| 전산화(Digitization) | 1997~2001 | EDI |
| 디지털화(Digitalization) | 2022~2024 | 프로세스 전자화·자동화 |
| **디지털전환(DX)** | **2025~** | AI·빅데이터·블록체인·클라우드, 차세대 나라장터 |

- **차세대 나라장터**(2023 구축·2025 본격): **온통조달**·**조달데이터허브**·**API 개방** — DX 핵심
- **하도급지킴이**(2013~) — 지원 프로세스
- 2002 **전자정부 11대 과제** 중 하나
- 국제 수상: UN e-Procurement Best Practice(2004), OECD NFAR, WCIT Global IT Excellence(2006) 등
- **2024말**: 이용자 67.4만(개통 대비 7.3배), **거래 156.8조**(4.3배) — 나라장터 **거래** 규모

**핵심 vs 지원 프로세스**

| 핵심(Main) | 지원(Sub) |
|-----------|----------|
| 발주, 전자입찰, 전자계약, 계약이행, **종합쇼핑몰** | 업체등록, 보증, **목록정보**, 통합검색, 조달통계, 문서유통, 표준연계, 하도급지킴이 등 20여 개 |

- 모듈: 사용자등록, 구매요청, 전자입찰, 전자계약, 검사검수, 전자지불
- 지원: 종합쇼핑몰, 혁신장터, 벤처나라, 이음장터, 담합징후, 조달데이터허브, **누리장터**(2014~ 민간 발주)
- **2024말**: **공공조달 ~225.1조**(전체 규모), 수요기관 7.2만, 조달기업 60.2만, 연계 시스템 ~180개

#### 2.2 이용자·인증

- **수요기관**: 개인회원 → 수요기관 등록 → **GPKI/EPKI** (행정·교육 전자서명)
- **조달기업**: 개인회원 → 조달업체 등록 → 사업자용 공동인증서
- **입찰대리인**: 간편인증·금융인증·스마트나라장터 인증 → 법인 공동인증서로 전자서명

#### 2.3 발주·중앙조달

**업무 흐름**: 발주계획공개 → 계약요청 → 사전규격공개 → 입찰공고 → 입찰·개찰 → 전자계약 → 검사검수 → 대금지급

**중앙조달 의무 (국가기관)**

| 대상 | 기준 |
|------|------|
| 물품·용역 | **1억 원 이상** |
| 종합공사 | 추정가격 **30억 원 이상** |
| 전문공사 | 추정가격 **3억 원 이상** |
| 단가·제3자단가·**MAS** | **금액 무관** (종합쇼핑몰) |

- **지방자치단체·교육기관**: MAS 등 조달청 물품은 의무, 그 외 2008~ **자체조달 가능**
- **공기업·준정부**: 중소기업자간 경쟁제품 **고시금액 이상** → 중앙조달. 입찰비리 시 **2년** 전체 위탁
- 모든 입찰공고 **나라장터(지정정보처리장치) 필수**. 2,000만 원 이상 **수의계약** → 나라장터 견적 경쟁
- **중앙조달 요청 비중**(2024): 법상 자체조달 가능해도 실무상 **의무 중앙조달 48.4%** vs **비의무(자체 등) 51.6%**

**소액수의계약**

| 구분 | 금액 | 방식 |
|------|------|------|
| 1인 견적 | 2,000만 원 이하 (여성·장애인·사회적기업 **5,000만 원**) | 견적 1건+예산 이내 |
| 2인 이상 견적 | 2,000만 원 초과~1억 원 (여성 등 **5,000만 원** 초과) | 전자 견적 경쟁 |

**MAS 2단계경쟁**: 일반 **5,000만 원↑**, 중소기업자간 경쟁제품 **1억 원↑**. 5억 원↑ → 제안공고·5개사(2개사 가능) 제안경쟁

#### 2.4 전자입찰·계약·지불

**적격심사낙찰 10단계**: 입찰공고 → 입찰준비 → 기초금액 공개 → 입찰서 제출 → **복수예비가격 15개** → 개찰 → 결과 확인 → 적격심사 → 심사 결과 → 낙찰자 선정

- **예정가격**: 15개 중 업체 2개씩 선택 → **최다 선택 4개** 산술평균 (차세대 나라장터)
- **전자계약 6단계**: 초안 준비 → 검토 → 응답서 → 접수 → 보증서 → 최종 발송

**전자계약 주요 효과** (3회 10번 함정)
- 계약 **책임성 강화** · 문서 **위·변조 방지** · 계약 체결 **기간 단축**
- **계약 자유 원칙 확대**는 효과가 **아님** — 표준화·통제 **강화** 쪽(오답)

**전자입찰서 제출·취소** (3회 26번 · ch06 교차)
- **동일 PC 1건**만 제출 · 제출 후 **교환·변경·취소 불가**
- 중대 오류 → **취소 신청** 가능하나 해당 입찰 **재참가 불가**(재공고·재입찰 시 예외)
- 부속서류 암호·바이러스 → **미제출** 간주

- **전자지불 6단계**: 검수요청 → 검수 → 결과서 → 대금청구 → 지급결의 → 계좌이체

**정보 공개 7단계**: 발주정보 → 사전규격 → 입찰공고 → 기술평가 → 개찰 → 계약·변경 → 대금지급

#### 2.5 입찰공고·가격유형

**공고기간 (국가계약법 시행령 §35, 지방 동일·소액 안내 3일은 토·일 포함 차이)**

| 유형 | 기간 |
|------|------|
| 협상(물품·용역) 일반 | **40일** |
| 협상 긴급 | **7일** |
| 공사 10억 미만 | 7일 / 10~50억 15일 / 50억↑ **40일** |
| 재공고 | 단축 5일·일반 10일 |
| 소액수의 안내 | **3일** (토·일 포함) |

- 입찰서 접수: 마감 **48시간 전**부터 / 개찰: 마감 **1시간 후** 이후

**금액 5종**

| 명칭 | 설명 |
|------|------|
| 사업금액 | 발주계획 예산 |
| 추정가격 | 사업금액 − 부가세 (국제입찰 판단) |
| 기초금액 | 예정가격 산정 기준 (거래실례·원가 등) |
| 복수예비가격 | 기초금액 ±**2%**(국가) / ±**3%**(지방) 내 **15개** |
| 예정가격 | 가격평가·계약 기준 |
| 낙찰하한가 | 예정가격 × 낙찰하한율 |

**예정가격 3유형**: ①**비예가**(1인견적·개산·기술제안·협상) ②**단일예가**(수의·전자시담) ③**복수예가**(대부분 입찰)

**소액수의 낙찰하한율 (국가계약법)**: 물품·용역 **88%**(단순노무 90%), 공사 **89.745%**. 2025.5.1(지방)·7.1(국가 공사) **87.745%→89.745%** (+2%p). 지방 2,000만↓ 1인견적 **90%**

### 제3절 연계 계약관리 지원시스템

#### 3.1 종합쇼핑몰 (2006~)

- URL: shop.g2b.go.kr — **e-Catalogue + e-Purchasing**, MAS 기반 디지털 마켓플레이스
- 대상: 조달청 **단가·제3자단가·MAS** 물품·용역
- 절차: 검색 → 장바구니 → 조달요청(주문) → 납품요구 → 납품
- **MAS**: 2인↑ 계약, 품질·다양성·선택권. 대상 — 규격확정·상용화, 연간납품 3,000만↑ 업체 3사↑, 단가계약 가능 등
- MAS 계약 **9단계**: 구매계획 → 공고 → 적격성평가 → 가격협상 → 계약 → 상품등록

#### 3.2 혁신장터 (2019~)

- 목적: 혁신제품·시제품 지정·판로 (전략적 공공조달)
- **직접 계약 기능 없음** → 국가계약법 시행령 §26①⑤**사**목 수의계약 또는 조달청 중앙조달
- MAS·제3자단가 등록 시 → 종합쇼핑몰 납품요구 가능
- 지정 후 **3년** 수의계약 (연장 최대 3년). 2025~ 유형 **2개**(유형1=FT1+3 통합, 유형2=FT2)
- **혁신구매 목표제**: 기관 물품구매액 **1%**
- 등록: 입찰참가자격 → 물품식별번호 → 전용몰 등록

#### 3.3 벤처나라 (2016~, 조달사업법 §25)

- 대상: 벤처·창업기업 직접생산·OEM 물품·용역
- **6년** 바로주문·견적주문. 추정가격 **2,000만↓**(여성·장애인·사회적기업 **5,000만↓**)
- 지정 7단계: 추천기관 추천 → 신청 → 심사 → 발표 → 등록 → 지정증서

#### 3.4 디지털서비스몰 (2022~)

- 클라우드 등 **디지털서비스** 전문 (클라우드컴퓨팅법 §20③)
- 3유형: ①클라우드컴퓨팅서비스 ②지원서비스 ③융합서비스
- **수의계약**(국가계약법 §26①⑤**아**, 지방 §25①⑧**차**) + **카탈로그계약**(조달사업법 시행령 §16)

#### 3.5 이음장터 (2022~)

### 제4절 목록정보시스템

#### 4.1 물품목록제도·물품목록법

| 연혁 | 내용 |
|------|------|
| 1962 | 물품관리법 |
| 1968 | 미국 FSC 도입 |
| 1991 | **물품목록법** 제정 (1995 시행) |
| 2002 | 나라장터 + **UNSPSC** 채택 |
| 2006~ | FSC 병행 → UNSPSC 단일 기준 |

- **물품**: 동산 중 현금·유가증권 제외
- **조달청**: 법 §4 — 목록화·제도 운영 (목록화지침 훈령)
- **적용**: 국가·지방기관 보유·소요 물품 (군수품 원칙 제외, 2020~ 일반 군수품 조달청 이관분 적용)
- **목록화 4원칙**: 통일성·포괄성·상호배제성·단순성
- **나라장터 계약**: 세부품명번호 **필수** (없으면 원칙적 계약 불가)

#### 4.2 목록번호 체계

```
물품목록번호 = 물품분류번호(8자리) + 물품식별번호(8자리)
             = UNSPSC 대·중·소·세(각 2자리) + [국내 세부품명 2자리] → 세부품명번호 10자리
```

- **품목명**: [세부품명, 제조업체명, 모델명, 대표속성값]
- **FSC**(구): 7자리+품목4자리=11자리 (품목 단독 식별 불가)
- **유보코드**: UNSPSC 미분류 시 **99**부터 역순

#### 4.3 목록정보시스템 이용

- URL: goods.g2b.go.kr
- **5메뉴**: 지능형검색, 검색, 목록화요청, 공지, 이용안내
- 지능형검색: ~30분 지연 / 검색: **실시간**
- **품목등록**: 4단계, **8업무일** 이내
- **품명등록**: 5단계, **24업무일**(신기술·융복합 **17업무일**)
- **일괄발급**: 동일 세부품명 **10개↑** 품목

#### 4.4 품목 관리

**용도 구분**: 일반(1)·계약(3)·시설자재(4)·입찰공고(5)·물품관리(6)

- 제조업체명: **입찰참가자격 등록증 제조** 등록 필수 (공급업 등록만으로는 제조명 불가)
- OEM: 제조계약서·상표권 → **(주문자상표부착)** 표기
- 등급: 속성·이미지 품질 **1~3등급**
- 이미지: jpg/gif/png, **2MB↓**, 400×400px, 최대 5장

#### 4.5 최신화·국제화

- 품명·세부품명 변경 → **상품목록심의회**
- 품목: 담당자 검토 (10년 미사용·중대 흠결 시 삭제 가능)
- UNSPSC 신설·변경 → 심의회 → 영문 신청 → UNSPSC 관리 사이트 상정

## 시험 출제 포인트

1. **전자조달 = 공공조달 실행·관리의 핵심**. KONEPS 2002, E2E, 전자공공조달=전자정부조달
2. **소싱 vs 구매** 2유형, **4사분면** 지출유형과 기능 매칭
3. **역경매 미채택**, **종합쇼핑몰=e-Catalogue**, **FA≈MAS**, **DPS≈MAS**(EU vs 한·미)
4. **나라장터 3단계**: 전산화→디지털화→**DX(2025~)**
5. **핵심 5 vs 지원 20+**, 누리장터(민간), 하도급지킴이(2013)
6. **중앙조달 의무 금액**: 국가기관 1억/30억/3억, MAS 무관, 지방 2008~ 자체조달
7. **적격심사 10단계**, **복수예비가격 15→4개 평균**
8. **공고기간·48h·1h** 규칙, **±2%/±3%**, **예정가격 3유형**
9. **5개 연계몰** 특성·한도·계약방식 (혁신=수의 3년, 이음=사전계약 없음, 벤처=6년)
10. **MAS 2단계**: 5천만/1억, 중소 10% 인하 제한
11. **UNSPSC**, **10자리 세부품명**, **8+8 목록번호**, 물품목록법 1991
12. **품목등록 8일·품명 24일**, 제조등록 필수, 상품목록심의회

## 함정·비교형

| 함정 | 정답 쪽 |
|------|--------|
| 전자조달 = 단순 구매·지불만 | 소싱·공급망·E2E 전 과정 |
| 나라장터 = 조달청 전용 | **모든 공공기관** + 누리장터(민간) |
| 역경매 = 국내 표준 | **국내 미채택** |
| 혁신장터·벤처나라·이음 = 쇼핑몰처럼 즉시 계약 | 혁신=수의/중앙조달, 이음=협상·직거래, 가격=희망가 |
| 종합쇼핑몰 = 모든 물품 | **MAS·단가·제3자단가** 한정 |
| 물품목록 = 나라장터만 | **물품관리법·물품목록법** 기준정보, 조달·관리 공통 |
| FSC = 현행 단일체계 | **UNSPSC** (2006~ 단일, FSC는 구체계) |
| 지능형검색 = 실시간 | **~30분 지연** (검색 메뉴=실시간) |
| 조달기업 목록화 = 법 §7 직접 근거 | 실무 허용, **법적 근거는 기관 요청** 중심 |
| PMIS = 포털과 동일 | PMIS=관리·감사, **포털=실행·게이트웨이** |
| **156.8조(거래)** | **225.1조(공공조달)** | 156.8=나라장터 **거래** 4.3배 / 225.1=2024 **공공조달 전체** |
| **전자계약 효과** | 계약 자유 확대 | 책임성·위변조 방지·기간 단축 O / **자유 원칙 확대 X** |
| **전자입찰 취소** | 재참가 가능 | 취소 신청 가능하나 **해당 입찰 재참가 불가** |
| **중앙조달 48.4%** | **자체 51.6%** | 법상 자체 가능해도 **중앙 요청** 비중이 더 큼(2024) |
| **온통조달·허브** | 구 나라장터만 | 차세대 **DX**(2025~) — API·데이터허브 |

**플랫폼 구분**

- **사전 조달청 계약 O**: 종합쇼핑몰, 디지털서비스몰(선정·카탈로그)
- **사전 계약 X**: **이음장터** (자유 등록)
- **지정·등록제**: 혁신장터, 벤처나라
