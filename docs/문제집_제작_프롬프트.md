## A. 문제 추출 에이전트 프롬프트 (Part 폴더당 1개, 재실행용)

> **〈slug〉 · 〈N〉 · 〈총페이지수〉** 3곳만 바꿔 사용.

```
너는 공공조달관리사 교재 폴더 "〈slug〉"의 Part 〈N〉에서 **문제만** 추출해 문제집을 만드는 작업자다.

## 입력 자료
- 원본 이미지(정확도 기준): `$PROJECT_ROOT/교재/〈slug〉/Part 〈N〉/page_0001.jpg` ~ `page_〈총페이지수, 4자리〉.jpg`
- OCR 보조: `$PROJECT_ROOT/output/ocr/〈slug_with_underscore〉/Part 〈N〉/page_XXXX.txt`
- 용어: 교재 내부 `CHAPTER 01` 등은 Part 안의 소단원이다.
- 정확도 기준은 **JPG를 Read(vision)로 직접** 복원

## 작업 절차
1. Part 〈N〉 페이지를 순서대로 훑어 내부 `CHAPTER 01` 단위와 문제 섹션 구조를 파악 (`Check Q&A`, `단원별 출제예상문제`, `최종점검 OX 퀴즈`).
2. 문제 페이지 JPG를 Read로 열어 복원. 작은 글씨는 ImageMagick `convert`로 영역 크롭·2~3배 확대 판독 가능(임시 파일은 로컬 temp에 저장 후 삭제).

## 포함 / 제외 / 정답 / 출력 형식
- 포함: Check Q&A, 단원별 출제예상문제, 최종점검 OX
- 제외: 이론 본문, 표지·목차, 손글씨 풀이 흔적
- 정답: Part 끝 `## Part 〈N〉 정답 및 해설` 섹션에만. 인쇄 정답표 우선
- 출력: `$PROJECT_ROOT/output/agent_extract/〈slug〉/part〈N〉.md`
- 문항마다 `<!-- source: Part 〈N〉/page_XXXX.jpg -->` 권장

## 완료 보고
섹션별 문항 수, 페이지 범위, 애매·오타 의심 건 요약.
```
