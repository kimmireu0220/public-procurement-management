# 시험 모의 선별 프롬프트

`output/problem_book_final/` 문제 은행(3,325문항, **박문각**)에서 **실제 1회 시험 규모**로 모의 세트를 뽑을 때 사용한다.  
**지문은 박문각만** 쓸 수 있으나, **회차 주제·클러스터는 `output/standard_textbook/`(조달청 표준교재 핵심 정리)를 먼저** 본 뒤 그에 맞게 문제 은행에서 선별한다.

## 판단 원칙

- **시험 고정값:** [`시험_안내.md`](../시험_안내.md) — 필기 80(30/20/30)·실기 20 내외·합격 기준·시간.
- **운영 판단:** Part·유형 비중, 보강·실기 구성, 클러스터 배치 등 — **에이전트가 정함**. 문서에 운영용 숫자를 박지 않는다.
- **사용자 명시 지시**가 있으면(문항 수·유형·난이도 등) **그 지시가 우선**한다.
- **Check Q&A 과목당 20% 이하** — 실제 필기에 없는 유형의 **출제 품질 상한**(검수·A-3). 보강·실기 규모 판단과 별개.

## 주제 설계 → 문항 선별 (표준교재 우선)

필기·실기·통합·과목 단독 **공통**. 선별 전에 반드시 거친다.

| 단계 | 자료 | 담당 | 산출 |
|:---:|------|------|------|
| **A. 주제** | `output/standard_textbook/<과목>/` | 출제 에이전트 | 회차별 **장·절·클러스터·함정** 목록 (머릿속 또는 `출제_방향.md`) |
| **B. 선별** | `problem_book_final` + `agent_extract` | 동일 | A에 맞는 박문각 지문만 `_draft`·최종 문제지 |

**A단계에서 볼 것**

- `INDEX.md` — PDF 장·박문각 Part 대응·시험 한 줄 포커스
- 해당 `ch0N_제N장_*.md` — 「시험 출제 포인트」「함정·비교형」「숫자·기준·법령」
- (3과목) `INDEX.md` 빈출 클러스터 ↔ 장 표
- (4과목 실기) `INDEX.md` 실기 클러스터 ↔ 장 표

**B단계에서 할 것**

- A에서 잡은 주제와 **주제가 맞는** `problem_book_final` 문항만 검색·선별
- `agent_extract` 정답·선지 대조 — 표준교재·법령과 모순이면 **제외·교체**
- 박문각 Part는 표준교재 장과 **대응**시키되, 지문·정답의 최종 근거는 `agent_extract`(박문각) + 표준교재 정합

**하지 말 것**

- 표준교재 없이 문제 은행만 훑으며 「맞는 것 같다」로 채우기
- 표준교재 범위 밖 주제를 박문각에서 억지로 끼워 넣기 (애매하면 다른 주제로 교체)

## 출제자 품질 기준 (매 회차 필수)

모의고사 작업 시 **강사가 아니라 필기 출제자**로 행동한다. 목표는 “많이 풀게 하기”가 아니라 **실제 시험에서 판별력이 있는 80문항**을 만드는 것이다.

### 출제자가 지켜야 할 것

1. **정확성** — 지문·정답은 `problem_book_final`·`agent_extract`·`국가계약법`/`지방계약법`/`조달사업법`·**표준교재 핵심**과 **모순 없어야** 한다. 애매한 문항은 제외하거나 교체한다.
2. **시험자가 알아야 할 핵심** — 조달 실무·법제에서 반드시 구분·암기·판단해야 하는 내용 위주:
   - 입찰 방식·절차·보증금·해제/해지·낙성/요물·부정당업자
   - 발주계획·사전규격·시장조사·원가·협상·구매 전략
   - 계약 이행·검사·대금·변경·MAS·물가변동·지체상금·하도급
3. **단원(Part·Chapter) 골고루** — 한 단원·한 주제에 몰지 않는다. Intro `page_0002.jpg` **출제기준 (30)/(20) 표기** 비중을 참고해 각 Part에서 고르게 뽑는다.
4. **중복 금지** — 동일 지문·동일 주제(파레토·MAS 기준금액 등)·유사 쌍둥이 문항 금지.
5. **이전 회차와 겹치지 않기** — **모든** 이전 회차 `manifest.json`의 `stable_id`·동일 지문은 〈K〉회차에서 **재사용 금지**.
6. **유형 균형** — `단원별 출제예상` 중심, ㉠㉡·빈칸형 일부 포함, Check Q&A는 과목당 **20% 이하**. O/X는 필기에 없으므로 **제외**.

### 난이도·핵심 출제 (실전 수준)

모의고사는 **연습용 쉬운 퀴즈가 아니라** 실제 필기에서 변별력이 나는 문항으로 구성한다. 사용자가 「난이도 올려줘」「핵심만」이라고 하면 아래를 **우선** 적용한다.

#### 쉬운 문항 줄이기

- **줄일 것:** 한 줄 정의·「~란?」·단순 O/X 변환형·선지가 지나치게 명확한 문항.
- **늘릴 것:** 「옳지 않은 것」「가장 거리가 먼 것」「모두 고른 것(㉠㉡)」·**숫자·기준금액·비율·기한**·**절차 순서**·**A와 B의 차이/공통점**.
- Check Q&A: 상한 20%가 아니라 **과목당 10% 내외**를 목표(필요 시 0~2문항만).
- `단원별 출제예상` 비중 **70% 이상**, 그중 **함정·비교형**을 우선 선별.

#### 필수 핵심 클러스터 (회차당 골고루 — 동일 클러스터 2문항 이상 금지)

| 영역 | 반드시 다룰 내용 (예) |
|------|------------------------|
| 입찰·낙찰 | 2단계·동시입찰·적격심사·예정가격·재공고·수의 |
| 보증금 | 입찰·계약·이행·하자 **구분**·율·반환 |
| 계약 성립·효력 | 청약·승낙·낙성/요물·해제/해지 |
| MAS·조달제도 | 2단계·기준금액·다수공급자·카탈로그 |
| 이행·변경 | 물가변동·설계변경·Turn-Key·지체상금·검사·기성 |
| 2과목 절차 | 발주·사전규격·RFQ/RFP/ITB·협상·가격평가 |
| 윤리·제재 | 부정당업자·입찰무효·담합 |

이전 회차에서 **약점·오답**이었던 클러스터는 「배제」가 아니라 **다른 지문·다른 각도**로 재출제한다.

#### 체감 난이도 목표

- 잘 복습한 수험생이 **60~75%** 나오는 것이 적정. 연속 **85% 초과**면 정의형·Check 비중이 과다한 신호 → 〈K+1〉에서 난이도 상향.

### 채팅 응시 흐름 (〈K〉회차, 사용자 선호)

파일은 **80문항 한 세트**로 저장하되, **채팅에는 과목별로 나눠** 올린다. 답안 체크 부담을 줄이기 위함.

| 순서 | 트리거 | 채팅에 게시 | 문항 번호 |
|:---:|--------|-------------|----------|

**규칙**
- 각 배치마다 해당 과목 **문항 전부**를 채팅에 표시 (선지 줄바꿈).
- **정답·해설은 사용자가 해당 구간 답을 보내기 전까지 공개하지 않음.**
- 1과목만 풀고 답 제출 → 채점 후, 「다음 문제」 요청 시 2과목 게시.
- 80문항 완료 후 전체 채점·`필기_풀이.md` 작성 가능.
- 다음 배치 안내 문구 예: *「1~30번 풀고 답 보내주세요. 2과목은 ‘다음 문제’라고 하시면 31~50번 올려 드립니다.»*

## 반영 방법
1. 오답노트에서 〈K〉(필요 시 누적) 틀린 문항을 **개념 묶음**으로 정리 (풀이 §D와 동일한 묶음 기준)
2. 80문항 **대부분**은 평소처럼 Part·Chapter 균형, 이전 회차와 **다른** stable_id
3. 오답 묶음 대응 문항은 **균형·클러스터 상한을 깨지 않는 범위**에서, 에이전트가 **필요한 만큼만** — 같은 유형·난이도, **다른 지문** (`problem_book_final`에서 새로 선별). 문서에 고정 문항 수·비율 없음
4. 사용자 피드백 해석 (고정 키워드·고정 개수 아님):
