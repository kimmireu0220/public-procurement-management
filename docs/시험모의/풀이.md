# 시험 모의 풀이·해설 프롬프트

모의고사 **응시 후** 내 답을 제출했을 때 사용한다.  
출제 방법: [`선별.md`](선별.md)

**판단:** 보강 규모·추가 응용 여부는 **에이전트 판단** — 사용자가 문항 수·범위를 **명확히 지정**하면 그 지시가 우선한다.

절 구조: **A** 통합 채점 · **C** 3과목 단독 · **C-1** 1과목 · **C-2** 2과목 · **B** 금지 · **D** 보강 · **E** 다음 회차 — **부록** 요청 예시 (`선별.md`의 **A-3** 교차검수·§C 체크리스트와 번호가 다름).

## 역할 분리

| 단계 | 프롬프트 | 산출물 |
|------|----------|--------|
| 출제 | `선별.md` | `필기_모의_문제.md`, `필기_모의_정답.md` |
| 풀이·해설 | **이 문서** | `필기_풀이.md`, `필기/통합/오답노트.md` 또는 `필기/{N}과목/오답노트.md` 갱신, (재응시 시) `풀이/필기_시도N_*.md` |

## 답안 붙여넣기 → 채점 (CBT · 채팅 공통)

CBT 종료 화면 **「답안 복사」** 문자열을 채팅에 붙여넣으면, 별도 「채점해줘」 없이도 **채점·파일 갱신**을 수행한다.

| 단계 | 사용자 | 에이전트 |
|:---:|--------|----------|
| 1 | CBT 응시 후 답안 문자열 붙여넣기 | `manifest`·`필기_모의_정답.md`로 채점 (최신 회차·과목 단독이면 `docs/{N}과목/cbt-meta.json` 회차 참고) |
| 2 | (동일 메시지) | **채팅:** 점수 요약 + 틀린 문항 **상세 해설** |
| 3 | (동일 메시지) | **파일:** `필기_풀이.md`(기본 형식) · `필기/{통합\|N과목}/오답노트.md` 틀린 행 **추가** |
| 4 | (선택) 「커밋」「푸시」 | 그때만 git commit·push — **답 제출만으로 커밋하지 않음** |

**답안 형식:** `1② 2④ 3④ … 30③` (CBT 복사 문자열과 동일 · 통합 80문항도 동일 패턴)

**회차·과목 판별**

| 유형 | 답 범위 | 풀이 파일 | 오답노트 |
|------|---------|-----------|----------|
| 통합 필기 | 1~80 (또는 과목별 분할 제출) | `필기/통합/〈K〉회차/필기_풀이.md` | `필기/통합/오답노트.md` |
| 3과목 단독 | 1~30 | `필기/3과목/〈K〉회차/필기_풀이.md` | `필기/3과목/오답노트.md` |
| 2과목 단독 | 1~20 | `필기/2과목/〈K〉회차/필기_풀이.md` | `필기/2과목/오답노트.md` |
| 1과목 단독 | 1~30 | `필기/1과목/〈K〉회차/필기_풀이.md` | `필기/1과목/오답노트.md` |

### `필기_풀이.md` 기본 형식 (통합·과목 단독 공통)

**맞은 문항 전부 상세 블록 금지.** 참고: `필기/3과목/8회차/필기_풀이.md` · `필기/2과목/2회차/필기_풀이.md`

1. 제목 · 응시일 · 점수 한 줄
2. **요약** — 맞음/틀림 수 · 취약 키워드 (선택)
3. **전 문항 요약 표** — 번호 · 결과 · 내 답 · 정답 · 키워드
4. **틀린 문항만** `## N. ❌` 짧은 블록 (내 답·정답·1~3문장)
5. (해당 시만) `## 정답 정정`

**상세 해설** — **채팅에만** (틀린 문항 위주, 맞은 문항은 생략 가능).

- 3과목 단독 〈K〉 예: `output/mock_exam/필기/3과목/1회차/` — 문항 **1~30**

## A. 필기 모의 풀이 (통합 80문항)

작업 시 **〈K〉=회차**만 교체한다.

```

### 과목별 분할 응시·채점
- 1과목(1~30)만 제출 → 해당 구간만 채점·오답 요약 (정답지 전체 공개 금지)

## 참고 파일 (채팅에 첨부)
- `output/mock_exam/필기/통합/〈K〉회차/필기_모의_문제.md`

## 채점 규칙
- 정답지 기준으로 O/X 판정
- 복수선택·㉠㉡형: 정답지와 동일해야 정답 (부분 점수 없음)
- 내 답 누락·형식 오류: 해당 문항 오답 처리 후 풀이에 명시
- 과목별 점수: 1과목(1~30), 2과목(31~50), 3과목(51~80) 각각 집계

## 해설 원칙
- **채팅:** 틀린 문항 **상세 해설** (정답 근거 + 내가 고른 선지 함정). 맞은 문항은 요약만 또는 생략.
- **`필기_풀이.md`:** 위 「기본 형식」만 — **틀린 문항만** 짧은 블록. 맞은 문항은 표에만.
- 수험서·문제 원본 범위 안에서 설명. 불확실하면 "원본 확인 필요"
- 정답지의 한줄 요약을 그대로 복붙하지 말고 풀어서 설명 (채팅)
```

## B. 하지 말 것

- 응시 전 정답·풀이 파일 내용을 미리 알려주기
- `필기_풀이.md`에 **맞은 문항까지** 1~N 전부 상세 블록 작성 (표 + 틀린 것만 짧게)
- 오답노트에서 이전 회차·시도 기록 삭제·덮어쓰기
- 사용자가 요청하지 않았는데 **git commit·push**

## 부록. 사용자 요청 예시

- CBT **답안 문자열만** 붙여넣기 → 채점 + `필기_풀이.md` + 오답노트 **자동 갱신** (상단 「답안 붙여넣기」절)
- `1② 2④ … 30③` (3과목 9회차 등 — 회차 생략 시 `cbt-meta.json` 참고)
- `〈K〉회차 필기 답 채점해줘: (80개 답)` — 통합, 과목별 분할 가능
- `3과목 단독` / `1과목` / `2과목` — §C · §C-1 · §C-2
- 커밋·푸시는 사용자가 **명시할 때만**
- `다음 문제` (2과목 31~50 또는 3과목 51~80 채팅 게시)
- `오늘 틀린 문제 한 줄 요약해줘`
- `틀린 부분 응용 문제 내줘` (§D 형식·정답 비공개 — 개수는 요청에 있으면 따르고, 없으면 에이전트 판단)

## D. 오답 보강·응용 퀴즈 (채팅 · 즉시 보강)

모의고사·보강 퀴즈 응시 **후** 틀린 주제를 짚을 때 사용한다.  
단순 암기 OX보다 **사례·판단형 응용**을 우선한다.  
**다음 회차 80문항 출제**와 별개 — 보강 퀴즈 주제를 이후 모든 모의에 고정 반영하지 않는다 (`선별.md` §A-0).

### 언제 내는가
- 회차 채점 후 오답 **한 줄 요약** 제공
- 사용자가 보강을 요청하거나, 강사가 **필요하다고 판단**할 때 — 아래 §「오답 묶음」으로 **규모를 정한 뒤** 제시 (문서에 정해진 총 문항 수 없음)
- 같은 **개념 묶음**에서 또 틀리면 **추가 변형** — 맞출 때까지 무한 반복하지 않음

### 오답 묶음 → 보강 규모 (에이전트 판단)

**원칙:** 총 문항 수·묶음당 문항 수는 **에이전트가 그때그때 정한다.** 문서·코드에 보강용 숫자(하한·상한·비율)를 박지 않는다. 사용자가 **명확히 지정**하면 그 지시가 우선한다.

1. **입력:** `output/mock_exam/필기/통합/오답노트.md` 해당 회차 행 + `필기_풀이.md` 오답 포인트  
2. **묶기:** 틀린 문항을 **개념 단위**로 합친다 (오답 문항 수 ≠ 보강 문항 수).  
   - 같은 개념에 여러 번 틀렸거나 헷갈림이 깊으면 그 묶음을 **더 깊게**  
   - 단순 실수·한 번만 틀린 것은 **짧게**  
3. **총량:** 묶음 수·깊이·남은 시간·사용자가 「N문항만」「오늘은 짧게」 등으로 준 말을 보고 정함. 한 번에 다 못 다루면 나머지는 다음 보강 세션 또는 〈K+1〉 출제로 미룸  
4. **제안:** 문항을 내기 **전에** 「이번엔 ○○ 묶음 중심으로 N문항」처럼 **짧게 안내**할 수 있음 (N은 상황 판단, 고정값 아님)  
5. **우선순위:** (a) 같은 묶음에서 반복 오답 (b) 당회차 다수 오답 묶음 (c) 사용자가 채팅으로 짚은 개념 — **고정 주제 리스트 아님**  
6. **출처:** 지문·정답은 `problem_book_final`·`agent_extract`와 모순 없게. 응용형은 **새 상황**으로 재구성 가능

### 문항 구성 원칙
1. **응용형** — 「A 기관이 ~했다」 등 **상황+판단**
2. **함정·비교** — **이번 오답 묶음** 또는 해당 문항 `source` Part와 연관된 함정만 (보증금 종류 구분, 절차 순서 등). 시험 전체 함정을 매 세트마다 전부 넣지 않음
3. **정확성** — `problem_book_final`·수험서와 모순 없는 선지만 사용
4. **중복** — 한 세트 안에서 같은 묶음을 **다른 각도로 반복**할 때는 학습 목적에 맞게만 (과도한 반복 금지)

### 채팅 표시 형식 (필수)

```markdown
### 1번
지문 …

① 선지
② 선지
③ 선지
④ 선지

---

### 2번
…
```

- 문항 번호는 `### N번` + `---` 구분
- 선지는 **항목마다 줄바꿈**
- 하단에 답 입력 예시만: `예: 2 3 4 3 3 3 3`

### 정답·해설 공개 규칙
- **사용자가 답안을 보내기 전까지 정답표·해설·정답 행 열지 않음**
- 답안 수신 후: 문항별 O/X 표 + 틀린 것만 1~2문장 오답 포인트
- 미제출 문항은 「미응답」으로 표시하고, 나중에 이어서 채점

### 완료 보고 (답 제출 후)
- 점수 n/N
- 틀린 문항: 번호·한 줄 이유
- (선택) 추가 응용 제안 — **아직 약한 묶음**만, 사용자가 원할 때만 (개수는 에이전트 판단)
